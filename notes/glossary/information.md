# Information

## Definition

Information is interpreted and context-dependent content, metadata, or signal that helps someone understand, decide, or act.

## Why It Matters

This definition is practical for information architecture because it avoids three common mistakes:

- treating raw data as if it were already meaningful
- treating information as identical to internalized knowledge
- treating only content objects as information while ignoring metadata, labels, provenance, and status signals

In IA work, the test is not whether something exists, but whether it can help a user find, distinguish, understand, trust, or use what they need in a given context.

## Key Characteristics

- It depends on interpretation. A record, field, or page is not useful information unless a person or system can make sense of it.
- It depends on context. The same signal may be informative in one task and noise in another.
- It includes more than content. Labels, timestamps, ownership data, relationships, and state can all function as information.
- It is action-relevant. In this workspace, information matters because it affects understanding, decisions, navigation, retrieval, coordination, or memory.

## Practical Heuristic

`Focus, Meaning, Action` is a useful test for whether something is functioning well as information:

- Focus: does it direct attention to what matters?
- Meaning: does it help the user interpret what they are seeing?
- Action: does it help the user decide or do something?

This works better as a heuristic than as the core definition. The definition says what information is. The heuristic helps evaluate whether a piece of content, metadata, or signal is actually useful in practice.

## Examples

- A runbook is information because it helps responders act during an incident.
- A service ownership field is information because it helps route questions and accountability.
- A timestamp on a document is information because it changes how much the document should be trusted.
- A dashboard panel is not automatically useful information if its meaning is ambiguous or disconnected from action.

## Related Concepts

- Data: raw values or records without sufficient interpretation
- Knowledge: understanding that has been internalized and can be applied
- Documentation: one common container for information, but not identical to information itself
- Metadata: descriptive or structural information about other information
